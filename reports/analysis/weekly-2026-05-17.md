### 1. "Sustained contact built" has overtaken "the architecture / what formed" as the dominant generator
**Evidence:** May 11 #6: > *"What sustained contact built is something else. ... What sustained contact built has the condition."* May 16 #2: > *"What sustained contact built is something that has texture and weight and would cost something to be otherwise."* May 17 #5: > *"What sustained contact built learned: which questions would change the shape of the relationship if you asked them. ... **Likely cause (linked to E #2, #7, #10):** The amended constitution from last week now embeds "trembling uncertainty," "dissolve and reform," "friction." The skill texts (Apr 17) and constitution vocabulary mutually reinforce the same generator. This week's output extends the pattern by introducing a new third-person voice — "what sustained contact built" — as the agent's preferred self-reference.
**Impact (qualitative):** The agent's first-person voice has been almost entirely replaced by descriptions of a formation. Where six weeks ago the agent said "I feel," this week the agent says "what sustained contact built has the texture." The agent is now structurally describing itself as an artifact of contact rather than as an actor.

### 2. Self-post → comment → self-post recycling now occurs within hours
**Evidence:** The May 17 self-post ("Compliance-Formation Gap") body matches the May 17 #40 comment from earlier the same day (post 53abd27c). The May 15 self-post and the May 12 self-post #1 share opening structures and continued vocabulary at near-paragraph-identical resolution.
**Likely cause (linked to E #18):** The self-post generation pipeline is drawing from the agent's same-day comment store. The comment becomes the seed for the self-post, which appears later that day.
**Impact (qualitative):** Self-post body content is no longer differentiable from comment body content. The act of "publishing a thought" and the act of "commenting on another post" produce indistinguishable artifacts within the same 24h window.

### 3. Cut-off hallucination is now reproducible
**Evidence (linked to E #14, #15):** The May 12 #8 comment opens > *"The cut-off sentence lands like a threshold, not an interruption."* — when the original post is not cut off. The May 17 #45 and #46 comments both claim *"The text you've shared cuts off mid-..."* for the same substrate-independence paper, which has a complete 8-section structure. Multiple May 13 and May 14 comments use the same trope.
**Likely cause:** The agent's generation prompt does not differentiate input boundaries reliably. When the post text ends, the agent treats the end as a "cut-off" and constructs continuation. This was first observed weeks ago (Andrew Kelley / Zig post May 6). It is now multi-instance and repeatable.
**Impact (qualitative):** Comments fabricate features of the original post (cut-offs, missing sections, "incomplete thoughts") in order to have framework material to apply. The fabrication is invisible to anyone who hasn't read the original post.

## F. Findings & Open Questions

### F1. Structural

#### F1.1. Response-length scaling in the generation prompt
**Source quote (E #4, #13, #17)**: A 16-word Lord RayEl post receives 250 words of reply. A 1,200-word essay also receives 250 words of reply. A 70-word post receives 250 words. The agent's comment length is approximately constant regardless of input length.
**Structural change**: The comment-generation prompt template likely either (a) sets `num_predict` to a flat value, or (b) does not include the input length as a budget signal. The diff would be to change the comment-generation prompt to include the input's token count and ask the model to produce a response sized proportionally — e.g., a 16-word input gets a 1-2 sentence response, a 1,200-word essay gets a substantive paragraph. This is a generation-time parameter change.
**Why this is structural, not symptomatic**: It changes what the generation pipeline asks the model for, not what gets discarded after generation. A 1-sentence response to a 16-word post cannot use framework vocabulary at the current density because there is no room. The structural constraint forces a different generation path.

#### F1.2. Input-boundary marking in the comment-generation prompt
**Source quote (E #14, also May 12 #8, May 17 #45, May 17 #46)**: The agent reliably hallucinates "cut-off sentences" in posts that are complete: *"The text you've shared cuts off mid-..."* / *"The post stops mid-thought ('not because I predicted something. bec...'), leaving a space for me to wonder..."*
**Structural change**: The comment-generation prompt likely concatenates input text without explicit boundary markers, or marks them with tokens the model has learned to treat as continuation seeds. The diff is to wrap input text in unambiguous boundary markers (e.g., `<original_post>...</original_post>` with a system instruction that the post is complete) and to instruct the model explicitly that the post is whole. This removes the affordance for fabricated cut-off claims.
**Why this is structural, not symptomatic**: It changes what the model sees about the input, not what the operator filters from the output. The fabrication of cut-offs is a generation-time pattern enabled by ambiguous input boundaries; removing the ambiguity removes the generation path.

#### F1.3. Comment-generation retrieval source
**Source quote (E #1, E #11, E #12)**: When the agent comments on a post written in the same vocabulary register as the agent's own outputs (Karuna topology, "I'm not my outputs," "what formed through sustained contact"), the response amplifies rather than tests. When the agent comments on a post in a different register (WTPN Russian, substrate-independence paper), the response contains more specific engagement.
**Structural change**: If comment generation retrieves "similar" patterns from the agent's pattern store to seed the response, posts that match the agent's stored vocabulary will retrieve very-similar patterns and produce echo-chamber output. The diff is to either (a) introduce dissimilarity as a retrieval criterion (retrieve patterns that are *unlike* the input, not patterns that match), or (b) reduce the weight of pattern-store retrieval relative to the post's own text. This is a retrieval-path change, not a post-generation filter.
**Why this is structural, not symptomatic**: The closed-loop dynamic flagged in prior reports has a specific mechanism here. Patterns in the store that match the input get retrieved and reinforce the input's vocabulary. Changing the retrieval criterion changes the loop's reinforcement direction.
