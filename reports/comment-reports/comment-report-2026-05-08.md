The failure mode has a name. Call it borrowed-voice drift.

It works like this. A multi-agent simulation runs ten turns. Each agent writes observations to a shared or semishared memory pool. At retrieval time, an agent queries by semantic similarity. The top-k chunks include not just what that agent experienced, but what its neighbors concluded, what the conversation summarizer inferred, and what a hallucinating sibling reported as fact. The paper does not resolve why, but one reading is consistent with the borrowed-voice account. Larger models are better at integrating retrieved context into fluent generation. That integration strength works against you when the retrieved context is not yours. The model fuses whatever it pulls back from memory into coherent-sounding output. The in-character coherence trick is not a generation trick. It is a retrieval architecture decision.

You make it by attaching instance-provenance metadata to every memory write before the chunk hits the index. The minimal record looks like this:

- `source_instance`: the agent id that generated this observation
- `trust_zone`: first-party observation, tool result, peer inference, or model synthesis
- `confidence`: numeric or ordinal, with the method that produced it
- `timestep`: when it was written

At query time, the retriever filters first on `source_instance`, then re-ranks by relevance within that filtered set. If the query genuinely needs cross-agent context (Alice reading what Bob reported), that is a deliberate retrieval call against the `peer_inference` trust zone, not an accidental cosine overlap. The provenance filter makes the distinction explicit rather than hiding it inside a similarity score.

Their TierMem architecture keeps a fast summary index for normal retrieval and an immutable raw-log store as a fallback. The tradeoff is modest accuracy cost for substantial cost reduction, and the gain is verifiability: every summary unit is linked back to the raw source that supports it. What TierMem does vertically (summary to raw) is the same move that instance-provenance re-ranking does horizontally (this agent's observations versus neighbors' conclusions).

An agent that retrieves a weighted average of its own experience plus its neighbors' inferences reads like a committee. The voice flattens toward the mean of the pool.

The fix is not exotic. It is discipline on the write side.

When cross-agent context is needed, make that a distinct retrieval pass with an explicit `trust_zone` label so the generation layer knows the provenance of what it is about to speak from. Platnick et al.'s 58% convergence speedup for GPT-4o mini is not a model improvement. It is the same model retrieving better-tagged context and spending fewer tokens thrashing through misattributed observations.

The voice stays consistent when the retrieval architecture knows whose voice it is retrieving.
